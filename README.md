# rag-document-qa

A document question-answering system built with Retrieval-Augmented Generation (RAG). Drop in your documents — PDF, Word, Excel, CSV, JSON, or text — ask a question in plain English, and get an answer grounded in those documents (not made up). Embeddings run **locally** (your files never leave your machine for the search step); only the final answer is generated by an LLM.

## What it does

1. **Loads** every supported file from a folder (PDF, Word, Excel, CSV, JSON, TXT).
2. **Splits** them into overlapping chunks so context isn't cut mid-thought.
3. **Embeds** each chunk into a vector (a list of numbers capturing its meaning) — locally, with `sentence-transformers`.
4. **Stores** the vectors in a FAISS index and saves it to disk (built once, reused after).
5. **Retrieves** the most relevant chunks for a question.
6. **Generates** an answer from those chunks using a Groq-hosted LLM.

---

## Tech stack

| Tool | What it does here |
|---|---|
| **LangChain** (community + text-splitters) | Document loaders + recursive chunking |
| **sentence-transformers** (`all-MiniLM-L6-v2`) | Turns text chunks into vectors — runs locally |
| **FAISS** (`faiss-cpu`) | Stores vectors and finds the nearest matches |
| **Groq** (`llama-3.3-70b-versatile`) | Generates the final answer from retrieved context |
| **python-dotenv** | Loads the API key from `.env` (keeps secrets out of code) |
| **pypdf / docx2txt / openpyxl + unstructured / jq** | Read PDF / Word / Excel / JSON files |

## Project structure

```
rag-demo/
├── src/
│   ├── data_loader.py    # load_all_documents() — reads PDF/Word/Excel/CSV/JSON/TXT into one list
│   ├── embedding.py      # EmbeddingPipeline — chunk documents, then embed chunks into vectors
│   ├── vectorstore.py    # FaissVectorStore — build/add/save/load + search the FAISS index
│   ├── search.py         # RAGSearch — ties store + Groq LLM together; retrieve → answer
│   └── app.py            # entry point / example usage
├── data/                 # your source documents live here
├── .env                  # GROQ_API_KEY=...  (never committed)
└── README.md
```

## Run it locally

```bash
# 1. Create the environment
conda create -n rag python=3.11 -y
conda activate rag

# 2. Install dependencies
pip install langchain langchain-community langchain-text-splitters langchain-groq \
            sentence-transformers faiss-cpu pypdf docx2txt openpyxl unstructured jq \
            python-dotenv numpy

# 3. Add your Groq API key (free tier) to a .env file in the project root
echo "GROQ_API_KEY=your_key_here" > .env

# 4. Put your documents in data/ , then run from the project root
python -m src.app
```

Or use it in code:

```python
from src.search import RAGSearch

rag = RAGSearch()                       # builds the index on first run, loads it after
answer = rag.search_and_summarize("What is the AI Risk Management Framework?")
print(answer)
```

## What I learned

Building this taught me how to actually build something real — how a **RAG pipeline fits together end-to-end**: why documents get chunked with overlap, what an embedding really is, how a vector store finds "nearest by meaning," and why you ground an LLM in retrieved context instead of letting it answer from memory. I also learned the practical side — keeping API keys out of code with `.env`, why `.py` modules beat notebooks for shipping, and how relative paths and imports behave in a package.
